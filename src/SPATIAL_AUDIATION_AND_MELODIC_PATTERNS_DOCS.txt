SPATIAL AUDIATION MODE & MELODIC PATTERNS — COMPLETE DOCUMENTATION
===================================================================

This document covers the Spatial Audiation ear training system and the
Melodic Patterns material generator. These are two separate modes within
the Music Trainer app, accessible via the mode dropdown in the header.


================================================================================
PART 1: APP STRUCTURE
================================================================================

The Music Trainer app is a single-page React application with no router.
Navigation is handled by two levels of conditional rendering:

  Level 1: MODE SELECTOR (header dropdown)
    Switches between 14+ major modes, each a self-contained tool.

  Level 2: TAB SELECTOR (within Spatial Audiation only)
    Switches between 7 ear training exercises that share controls.

MODES:
  ear-trainer          Spatial Audiation (default) — 7 ear training tabs
  melodic-patterns     Melodic Patterns — chord-melody material generator
  drum-patterns        Drum Patterns — rhythm permutation engine
  drill-response       Drill & Response — call-and-response exercises
  uncommon-meters      Uncommon Meters — odd time signature training
  chord-chart          Chord Chart — chord progression reference
  konnakol             Solkattu — South Indian rhythm syllable system
  note-entry           Quick Transcriptions — notation entry tool
  phrase-decomposition Phrase Decomposition — phrase analysis
  lattice              Harmonic Lattice — JI lattice visualization
  interval-browser     Interval Browser — interval reference tool
  microwave            Microwave — microtonal exploration
  temperament-explorer Temperament Explorer — EDO temperament analysis

All modes share:
  - Header with mode dropdown, Stop Audio button, Practice Log, Settings
  - Metronome strip (BPM, tap tempo, time signature, accent patterns)
  - Practice timer
  - Audio engine (Web Audio API, supports 12/31/41-EDO tuning)
  - localStorage persistence via useLS hook

State: section is stored in localStorage as "lt_app_section".
Each mode manages its own internal state independently.


================================================================================
PART 2: SPATIAL AUDIATION MODE
================================================================================

Spatial Audiation is the primary ear training mode. "Spatial" refers to
the Lumatone isomorphic keyboard visualization — you develop audiation
(inner hearing) by connecting sound to spatial patterns on the keyboard.

2.1 SHARED CONTROLS
--------------------

All 7 tabs share these controls, passed via sharedTabProps:

  TONIC (tonicPc): 0-11 pitch class selector. Sets the key center.

  LOWEST/HIGHEST OCTAVE: Range controls (1-7). Defines the pitch space.

  EDO: 12, 31, or 41 equal divisions of the octave.

  RESPONSE MODE:
    "Play Audio" — hear the sound, identify what it is
    "Show Target (Sing It)" — see the target on keyboard, sing it

  PLAY VOLUME: Global playback volume.

  KEYBOARD LAYOUT: Lumatone isomorphic keyboard visualization.
    Highlights active pitches during playback.
    Supports 12-EDO, 31-EDO, 41-EDO layouts.

  DRONE: Continuous tonic reference tone.
    Single — tonic only
    Root+5th — tonic + perfect fifth
    Tanpura — simulated tanpura drone


2.2 INTERVALS TAB
-------------------

PURPOSE: Identify intervals by ear.

PIPELINE:
  1. System plays two notes (ascending, descending, or harmonic)
  2. User identifies the interval
  3. Feedback: correct/incorrect with keyboard visualization

CONTROLS:
  - Direction: ascending, descending, harmonic, random
  - Interval pool: select which intervals to drill (m2 through P8+)
  - Difficulty: simple (diatonic), chromatic, microtonal (31/41-EDO)

The Lumatone keyboard shows the spatial distance between notes,
building the visual-auditory connection that defines spatial audiation.


2.3 CHORDS TAB
---------------

PURPOSE: Identify chord qualities and practice chord voicings.

TWO SECTIONS:

  ISOLATED CHORDS:
    Plays individual chord progressions using formula patterns:
      "I X I" — tonic, target chord, tonic
      "ii/X V/X X" — secondary approach to target
    User identifies the target chord quality.

  FUNCTIONAL HARMONY:
    Generates looping chord progressions using a harmonic graph.
    Chords follow common-practice voice-leading (T→PD→D→T).
    Bass line + melody layers can be added (polyphonic realization).

CONTROLS:
  - Tonality bank: Major, Minor, Dorian, Mixolydian, etc. (15+ tonalities)
  - Tonality levels: Primary → Diatonic → Secondary Dominants → Borrowings →
    Secondary II-Vs → Tritone Subs (graduated complexity)
  - Chord types: triads, 7ths (maj, min, dom7, dim, aug, halfdim7, etc.)
  - Chord type mode: "any" or "by-third" (filter by third quality)
  - Voicing patterns: 60+ patterns across 6 groups
  - Extensions: 7th, 9th, 11th, 13th with tendency controls
  - Register mode: Fixed, Random Bass Octave, Random Full Register
  - Loop length: 2-12 chords per loop
  - Chord duration and gap timing


2.4 MELODY TAB
---------------

PURPOSE: Identify and sing back melodic fragments.

PIPELINE:
  1. System plays a melodic fragment (2-8 notes)
  2. User identifies or sings back the melody
  3. Feedback with keyboard visualization

CONTROLS:
  - Melody length: number of notes
  - Scale context: diatonic, chromatic, microtonal
  - Contour: ascending, descending, arch, random
  - Interval content: stepwise, leaps, mixed


2.5 JAZZ CELLS TAB
--------------------

PURPOSE: Practice common jazz melodic vocabulary cells.

Plays short idiomatic jazz patterns (2-4 notes) over chord contexts.
Cells include: enclosures, approach patterns, guide-tone lines,
bebop scale fragments, chromatic passing tones.

User identifies the cell type and its relationship to the chord.


2.6 PATTERNS TAB
-----------------

PURPOSE: Pattern recognition and reproduction.

Plays rhythmic or melodic patterns that the user must identify
or reproduce. Patterns increase in complexity.


2.7 CHORD DRONE TAB
---------------------

PURPOSE: Sustained harmonic context for free practice.

Plays a continuous chord voicing that the user can practice over.
Good for: improvisation practice, interval recognition in context,
singing exercises over harmony.

CONTROLS:
  - Chord selection from the tonality bank
  - Voicing pattern
  - Drone volume and sustain


2.8 MODE ID TAB
----------------

PURPOSE: Identify musical modes by ear.

PIPELINE:
  1. System plays a scale or melodic fragment in a mode
  2. User identifies the mode (Ionian, Dorian, Phrygian, etc.)
  3. Feedback with characteristic interval highlighting

CONTROLS:
  - Mode pool: which modes to include
  - Presentation: full scale, partial scale, melodic fragment
  - Reference: with or without tonic drone


================================================================================
PART 3: MELODIC PATTERNS MODE
================================================================================

Melodic Patterns is a SEPARATE mode from Spatial Audiation, selected
from the same mode dropdown. While Spatial Audiation is an ear training
drill (hear → identify → feedback), Melodic Patterns is a material
generator (configure → generate → curate → export → practice).


3.1 PURPOSE AND PIPELINE
--------------------------

Melodic Patterns generates chord-melody material for harmonic instrument
practice. It is the equivalent of Chaffee's sticking permutation books
for drummers, but for harmonic instruments.

PIPELINE:
  1. CONFIGURE — Pick EDO, tonality, harmony pool, melody categories,
     motion, approach/enclosure settings
  2. GENERATE — Create chord-melody segments
  3. CURATE — Lock segments, step through alternatives, browse chords
  4. LISTEN — Play with rhythm-aware timing, visualize on keyboard
  5. ITERATE — Reshuffle the variable side while fixed side stays locked
  6. SAVE — Presets, practice log, HTML export
  7. PRACTICE — Take material to your instrument


3.2 TWO PIPELINES
-------------------

CHORDS FIXED → MELODIES VARY:
  Fix a chord progression. Generate melody permutations over it.
  "Same Chord" mode: one chord, N different melodies.
  The core drill: internalize how different lines sound against one harmony.

MELODY FIXED → CHORDS VARY:
  Fix a melody. Find different chords that fit under it (reharmonization).
  "Chord Fit" slider: control consonance/dissonance of the pairing.
  The core drill: hear how the same melody changes character over
  different harmonies.


3.3 HARMONY SYSTEM
-------------------

14 harmony categories organized in 4 layers:

  DIATONIC:     Functional Harmony, Modal Interchange
  CHROMATIC:    Secondary Dominants, Secondary Diminished, Neapolitan,
                Tritone Substitutions, Chromatic Mediants
  EXTENDED:     Chromatic Diatonic (12 roots), Chromatic 31 (all roots)
  XENHARMONIC:  Subminor, Neutral, Supermajor

Progression modes:
  FUNCTIONAL — Markov chain with common-practice transition weights.
               Diatonic, modal, chromatic, and xenharmonic transitions
               all encoded. Xenharmonic chords substitute for their
               nearest diatonic relative at competitive weights.
  RANDOM — Uniform selection with soft common-tone preference.

Chord scoring (melody-first): 5-factor system evaluating coverage,
guide-tone match, voice-leading, functional probability, and identity
strength. Not just overlap filtering.


3.4 MELODY GENERATION
-----------------------

Four-pass architecture:

  PASS 1: ROLE ASSIGNMENT
    Metric weights from rhythm generator determine target positions
    (chord tones on strong beats) vs connector positions (passing tones).

  PASS 2: TARGET GENERATION
    Unified tension model combining:
      - Metric weight (40%): strong beats → stable notes
      - Phrase arc (30%): opening/closing → grounded, middle → colorful
      - Bias (30%): inside → chord tones, outside → tensions

    Stepwise constraint: exponential decay by perceptual interval size.
    Chord tones exempt — always reachable even in microtonal space.

    Cross-segment voice-leading: last note targets half-step resolution
    into next chord. First note targets guide tones (3rds/7ths).

    Duration-aware tension: long notes pull harder toward chord tones.

  PASS 3: CONNECTOR FILLING
    Five strategies selected probabilistically per gap:
      Scalar (4.0), Motivic (3.0), Enclosure (2.5),
      Approach (2.0), Free fill (2.0)

    fillSmooth walks toward anchors via scale steps.
    Boundary intervals checked to prevent "smooth walk ending in leap."

  PASS 4: OCTAVE PLACEMENT
    Contour-aware in all modes (including stepwise — gentle nudging
    within step-size intervals). Registral pull prevents drift.
    Leap recovery enforces contrary motion after large intervals.


3.5 NOTE CLASSIFICATION
-------------------------

Five categories based on accidental count + JI consonance:

  ct          — chord member (0 accidentals or in chord PC set)
  diatonic    — 0 accidentals, natural tensions (9, 11, 13)
  chromatic   — 1 accidental (b9, #11, b13)
  micro       — 2+ accidentals, approximates 7-limit or 11-limit JI ratio
  microTense  — 2+ accidentals, NOT near any low-limit JI ratio

The stable/tense microtonal split is based on odd-limit consonance
theory (Partch, Tenney, Erlich). 7-limit intervals (harmonic series
partials 7, 9, 14) and 11-limit intervals (neutral quality) are
classified as stable. Everything else with 2+ accidentals is tense.


3.6 RHYTHM INTEGRATION
------------------------

MelodicRhythm component generates two notation lines (chord + melody):
  - 6 styles: Straight, Syncopated, Triplet, Bossa, Clave, Displaced
  - Time signatures: standard presets + custom
  - Metric weights feed into melody generation (target assignment + tension)
  - Rhythm timing data feeds into playback (variable note durations)

Playback uses per-note durations derived from the rhythm pattern.
A dotted quarter rings longer, an eighth note is short.


3.7 FEATURES
--------------

PER-SEGMENT CONTROLS:
  - Lock (🔒): skip during reshuffle, keep segments you like
  - Step chord (↻): try another chord from the pool
  - Step melody (↻ melody): try another melody permutation
  - Browse: full chord browser filtered by harmony pool
  - Click melody blocks: highlight on keyboard
  - Click chord name: highlight all chord tones on keyboard

PRESETS:
  Save/load full configuration snapshots to localStorage.
  All settings captured: EDO, tonality, pipeline, harmony pool,
  melody categories, bias, motion, approach, enclosure, etc.

PRACTICE LOG:
  Log Session records timestamp, settings, duration, segment count.
  History view shows past sessions. Keeps last 100 entries.

EXPORT:
  Self-contained editable HTML with Print/PDF and Download HTML buttons.
  Content is contenteditable — annotate, edit chord names, etc.

KEYBOARD VISUALIZER:
  Lumatone isomorphic keyboard for the current EDO.
  Floating miniature appears when inline version scrolls out of view.
  Real-time highlight during playback.
  Click-to-highlight on melody notes and chord names.

KEYBOARD SHORTCUTS:
  Space: play all segments
  R: regenerate

PART 4: HOW THE TWO MODES RELATE
================================================================================

Spatial Audiation and Melodic Patterns serve different stages of the
same learning process:

  SPATIAL AUDIATION — RECOGNITION
    Hear sound → identify it → build the audiation connection
    Passive-to-active: the system presents, you respond
    Goal: develop inner hearing (audiation) mapped to spatial patterns

  MELODIC PATTERNS — PRODUCTION
    Configure constraints → generate material → study → practice
    Active-to-creative: you set the parameters, system generates options
    Goal: build vocabulary for improvisation and composition

The connection: Spatial Audiation trains you to HEAR intervals, chords,
and modes. Melodic Patterns gives you MATERIAL that uses those same
elements in melodic context over chord progressions.

A practice session might look like:
  1. Warm up with Spatial Audiation intervals tab (recognize intervals)
  2. Switch to Melodic Patterns (generate ii-V-I melodies with approach tones)
  3. Export the material, take it to your instrument
  4. Practice the generated lines over the progression
  5. Return to Spatial Audiation chord drone tab to improvise freely

Both modes share:
  - The same EDO tuning system (12/31/41)
  - The same audio engine
  - The same Lumatone keyboard visualization
  - The same chord type catalog from edoData.ts
  - The same pitch-class and interval infrastructure

They differ in:
  - Interaction model: drill (SA) vs generator (MP)
  - Complexity: SA has curated difficulty levels; MP has raw parameter control
  - Output: SA produces scores; MP produces exportable material
  - Rhythm: SA uses simple playback; MP integrates rhythm into generation
  - Melody: SA plays fixed patterns; MP generates algorithmically with
    4-pass architecture, contour shaping, voice-leading, and unified
    tension model


================================================================================
PART 5: EDO TUNING SYSTEM (SHARED)
================================================================================

Both modes operate within the same EDO framework:

12-EDO:
  Standard Western tuning. 12 equal divisions of the octave.
  All chord types: standard triads + 7ths (9 types).
  No microtonal categories (micro/microTense are empty).

31-EDO:
  Meantone temperament. 31 equal divisions. Step size: 38.7 cents.
  Diatonic structure: T=5, s=3, A1=2 steps.
  7 distinct third sizes: 7 (subminor), 8 (minor), 9 (neutral),
  10 (major), 11 (supermajor) steps.
  Extended chord catalog: 24 chord types (9 universal + 15 xen).
  Rich microtonal space: bb/## intervals, septimal and undecimal
  consonances accessible.

  Key JI approximations:
    5/4 major third: 10 steps (387¢, error +0.7¢)
    7/4 harmonic 7th: 25 steps (968¢, error -0.8¢)
    11/8 undecimal 4th: 14 steps (542¢, error -9.7¢)

41-EDO:
  More precise JI approximation. 41 equal divisions. Step size: 29.3 cents.
  Diatonic structure: T=7, s=4, A1=3 steps.
  7 distinct third sizes: 9-15 steps, including classic minor (6/5)
  and classic major (5/4) as separate from Pythagorean.
  Extended chord catalog: 40+ chord types.

  Key JI approximations:
    5/4 major third: 13 steps (380¢, error -6.3¢)
    7/4 harmonic 7th: 33 steps (966¢, error -2.8¢)
    11/8 undecimal 4th: 19 steps (556¢, error +4.8¢)

All three EDOs use the same infrastructure:
  - getEdoChordTypes(edo): complete chord type catalog
  - getFullDegreeNames(edo): diatonic naming with accidentals
  - getDegreeMap(edo): interval name → step mapping
  - getEDOIntervals(edo): standard interval shortcuts (M3, P5, m7, etc.)
  - Lumatone layout files for keyboard visualization
